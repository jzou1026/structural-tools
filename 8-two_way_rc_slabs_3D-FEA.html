Theory and Engineering Notes: SAFE-Lite 2D Shell Slab FEM

This section is written for engineers reviewing or modifying the program. It explains how the browser app turns an entered slab model into finite-element shell forces, design strip diagrams, and SAFE-style design tables.

Q4 Mindlin plate shell Sparse conjugate-gradient solver Column/wall spring supports M11 / M22 / M12 contours Design strip BMD/SFD Eq. 11.6 one-way shear
Scope: this page documents the current implementation. It does not change the solver, design equations, or results. Final engineering design should still be checked independently.
1. Program Workflow

The app follows the same conceptual workflow as a commercial slab finite-element model: define geometry and material properties, generate a shell mesh, solve the elastic model, recover element forces, integrate forces over design strips, and summarize design checks in tabular form.

Inputgeometry, slab, loads
MeshQ4 shell elements
SolveK · d = F
RecoverM, V, w, As
Design Tablesstrip and support checks
2. Geometry Model

The slab plan is defined using joints, rectangular slab areas, columns, walls, and design strips. The solver works in millimetres and Newtons internally, then reports engineering output in common slab-design units.

JointsID, x, y, z control geometry and support locations.
RectanglesEach rectangular slab area is meshed into Q4 elements.
ColumnsColumn dimensions define support stiffness and column-face design stations.
WallsWall lines are converted into distributed elastic support springs.
Design stripsStrip boundaries select which shell results are integrated into BMD/SFD.
For Example 12.6, the default strip X_GRID2 is the primary comparison strip along gridline 2.
3. Units and Sign Convention
  • Coordinates, slab thickness, effective depth, column dimensions, and mesh size are entered in mm.
  • Loads are entered in kPa, then converted to N/mm² during element load construction.
  • Element bending results are reported as kN·m/m.
  • Element transverse shear results are reported as kN/m.
  • Design strip BMD is reported as aggregate kN·m; design strip SFD is reported as aggregate kN.
  • Positive vertical displacement, w, is taken downward in the plate DOF convention.
4. Material and Plate Element Formulation

Each shell element is a four-node Mindlin-Reissner plate element with three degrees of freedom per node: vertical displacement and two rotations. The formulation includes bending and transverse shear deformation.

DOF per node = [ w, θx, θy ] Ec = 4500√f′c unless Ec override is entered G = Ec / [2(1 + ν)] Db = Ec h³ / [12(1 − ν²)] · [[1,ν,0], [ν,1,0], [0,0,(1−ν)/2]] Ds = (5/6) G h · I

The element stiffness is assembled from bending stiffness and shear stiffness. Bending terms use 2×2 Gauss integration, and shear terms use reduced 1×1 integration.

Ke = ∫ Bbᵀ Db Bb dA + ∫ Bsᵀ Ds Bs dA
5. Applied Gravity Load

The program includes slab self-weight, superimposed dead load, and live load. These are factored and applied as a uniform area pressure to the slab shell elements.

Self-weight DL = γc · h Specified DL = γc · h + SDL qULS = Df(γc · h + SDL) + Lf(LL) Internal pressure = qULS × 0.001 N/mm²

For the Example 12.6 defaults, the applied factored load is:

γc = 24 kN/m³ h = 0.180 m SDL = 1.44 kPa LL = 3.60 kPa qULS = 1.25(24×0.180 + 1.44) + 1.5(3.60) qULS = 12.60 kPa
6. Column and Wall Supports

Columns and walls are represented with elastic springs attached to mesh nodes. This gives the slab vertical and rotational restraint without explicitly meshing the column shafts.

Column vertical spring: kz = Ec A / Ltop + Ec A / Lbottom Column rotational spring: kθ = α Ec I / Ltop + α Ec I / Lbottom α = 4 for fixed far end α = 3 for pinned far end

Wall support springs are distributed along the wall line based on tributary wall length assigned to each support node.

7. Global Solution

The app assembles all element stiffnesses, support springs, and equivalent nodal loads into a global sparse system. The unknowns are the nodal vertical displacements and rotations.

K · d = F K = shell element stiffness + column/wall support spring stiffness d = global displacement/rotation vector F = equivalent nodal vertical load vector

The solver uses a conjugate-gradient iteration with diagonal preconditioning. The solver log reports node count, shell count, degrees of freedom, iteration count, residual, and total applied factored load.

8. Recovered Element Forces

After solving for nodal displacements, the program evaluates element-centre curvatures and shear strains, then recovers bending moments and transverse shear forces.

κx = dθx/dx κy = dθy/dy κxy = dθx/dy + dθy/dx γxz = θx + dw/dx γyz = θy + dw/dy [M11, M22, M12]ᵀ = Db [κx, κy, κxy]ᵀ [V13, V23]ᵀ = Ds [γxz, γyz]ᵀ
9. Contours and Hover Values
  • M11 is the local x-direction bending moment, reported in kN·m/m.
  • M22 is the local y-direction bending moment, reported in kN·m/m.
  • M12 is twisting moment, reported in kN·m/m.
  • V13 and V23 are transverse shear components, reported in kN/m.
  • The contour colour scale maps blue to the displayed minimum and red to the displayed maximum.
  • Hovering over a contour element reports the element ID, location, and M/V shell forces.
Contour results are element-centre values. Design strip tables are integrated strip values, so the numerical units and interpretation are different.
10. Preliminary Flexural Reinforcement Demand

The program uses a Wood-Armer-style transformation to convert twisting moment into directional design moments. Positive demand is treated as bottom reinforcement demand, and negative demand is treated as top reinforcement demand.

Direction 1 bottom: M1+ = max(0, M11 + |M12|) Direction 1 top: M1− = max(0, −(M11 − |M12|)) Direction 2 bottom: M2+ = max(0, M22 + |M12|) Direction 2 top: M2− = max(0, −(M22 − |M12|))

The required steel area is then calculated using the app's rectangular stress block routine for a 1000 mm design width.

α1 = 0.85 − 0.0015 f′c φc = 0.65 φs = 0.85 b = 1000 mm As = [B − √(B² − 4 A Mu)] / (2A) − As,existing
11. Design Strip Aggregation

Design strips convert shell forces per metre into aggregate strip forces. This is the step that converts a shell FEM result into a design-strip result similar to equivalent-frame output.

X-direction strip: Aggregate BMD = Σ(M11 · dy) Aggregate SFD = Σ(V13 · dy) Y-direction strip: Aggregate BMD = Σ(M22 · dx) Aggregate SFD = Σ(V23 · dx)

Only shell elements whose centroids lie inside the selected strip are included. The strip plots are then drawn in the same filled-diagram style used by the earlier Two Way Slabs app.

13. Table 2 — Bending Moment Design

Table 2 is generated from the selected design strip only. The program reports the span label, face-to-face span indication, design station, aggregate moment, required steel, and approximate bar spacing.

Top steel rows: Left Face (Top) = aggregate strip moment at left column face Right Face (Top) = aggregate strip moment at right column face Bottom steel row: Max in Span (Bot) = maximum positive aggregate strip moment between column faces
Column-face stations use the entered column dimension in the strip direction. For X-strips, column face offset is cx/2. For Y-strips, column face offset is cy/2.
14. Table 3 — One-Way Shear Design

Table 3 is also generated from the selected design strip only. The program evaluates aggregate strip shear at a distance dv from each column face and compares it to concrete shear resistance calculated using Equation 11.6.

Shear design stations: Left check = left column face + dv Right check = right column face − dv Equation 11.6: Vc = φc λ β √f′c bw dv β = max(0.05, 230 / (1000 + dv)) dv = max(0.9d, 0.72h) √f′c ≤ 8 MPa λ = 1.0
15. Table 4 — Two-Way Shear Design

Table 4 provides SAFE-like punching quantities for columns within the selected design strip. The values are estimated from the finite-element support spring reactions.

Vertical punching force estimate: Vf ≈ |column spring vertical reaction| − load inside critical perimeter Unbalanced moment estimate: Mps = √(Mx² + My²) Mx, My are recovered from column rotational spring reactions.
This is an engineering approximation intended to communicate the FE-to-design workflow. It is not a full replacement for a code punching shear module with perimeter trimming, openings, drops, capitals, stud rails, or all code-specific β/γv factors.

The default model represents the Example 12.6 slab layout and loading used during validation. The main comparison strip is X_GRID2, which corresponds to the design strip along gridline 2.

17. Engineering Limitations and Review Checklist
  • Results depend on mesh density. Finer mesh generally changes local peak values near supports.
  • Column and wall supports are spring idealizations, not full 3D solid/contact models.
  • Design strip values are based on centroid inclusion and numerical integration of shell resultants.
  • Two-way shear results are approximate FE reaction summaries and should be checked against governing code provisions.
  • Bar spacing values in the design table are approximate quick-design outputs, not final detailing.
  • Use contour plots, strip diagrams, and tables together. No single table should be treated as the complete design basis.
Engineer review required: confirm load combinations, support modelling, mesh convergence, span definitions, effective depth, material factors, minimum reinforcement, maximum spacing, punching perimeters, and final detailing requirements before issuing design.
